# 黄晓林角色卡 — 怎么保存

## 你要改的东西在「项目里」才有效

- **世界书**：改的是 `示例/黄晓林角色卡/世界书/*.yaml`，新条目要在 `index.yaml` 的「条目」里加一条并写上 `文件: 世界书\文件名`。
- **状态栏界面**：改的是 `示例/黄晓林角色卡/界面/状态栏/` 下的源码；构建后会自动复制到正则文件。

在酒馆里改世界书/变量，**不会**自动写回这些本地文件，所以导出时还是旧数据。

---

## 保存步骤（推荐一条命令做完）

在项目**根目录**（有 `package.json` 和 `tavern_sync.yaml` 的目录）打开终端，执行：

```bash
pnpm run 保存黄晓林
```

会依次做三件事：

1. **pnpm build**：编译界面并把最新的 `dist/.../index.html` 复制到 `正则/[界面]黄晓林DnD状态栏.txt`。
2. **sync bundle**：把 `示例/黄晓林角色卡/` 下的 index、世界书、正则等打成一个角色卡 PNG。
3. 在终端里打印出生成好的 PNG 路径。

生成的文件是：

- **示例/黄晓林角色卡/黄晓林角色卡.png**

---

## 最后一步（必须做，否则酒馆里还是旧的）

在 **SillyTavern 酒馆**里：

- 用「导入角色卡」或「替换当前角色卡」，
- 选择刚才生成的 **黄晓林角色卡.png**，

这样酒馆里用的才是最新世界书和最新状态栏。只打包、不重新导入，界面上看到的还是原来的数据。

---

## 分步执行（可选）

如果想分开做：

```bash
pnpm build                    # 只编译 + 更新正则文件
node tavern_sync.mjs bundle 黄晓林角色卡   # 只打包角色卡（用当前本地文件）
```

然后再在酒馆里重新导入 `示例/黄晓林角色卡/黄晓林角色卡.png`。

---

## 改完没效果？请逐项核对

1. **必须重新导入 PNG**  
   打包只会更新本地的 `示例/黄晓林角色卡/黄晓林角色卡.png`。酒馆里要用「导入角色卡」或「替换当前角色卡」选这个 PNG，**否则界面上仍是旧数据**。

2. **确认用的是刚打包的卡**  
   导入后请确认当前对话用的角色卡就是刚导入的那张（名字/头像一致），而不是别的卡或旧版本。

3. **若开了酒馆助手「实时监听」**  
   实时监听可能从**网络或本地 URL** 拉取界面，而不是用角色卡里嵌的 HTML。  
   - 若界面来自 **CDN（如 jsDelivr）**：你改本地代码不会生效，因为酒馆加载的是网上旧版本。  
   - 解决：在酒馆助手设置里把该界面的来源改为「使用角色卡内嵌」或指向你本机（例如 `http://127.0.0.1:8080/...` 且本地已用 `pnpm run dev` 之类在跑），或关闭该条目的「从 URL 加载」。

4. **浏览器缓存**  
   若以上都做了仍无变化，可对状态栏所在页面做一次**强制刷新**（Ctrl+Shift+R 或 Ctrl+F5）。
