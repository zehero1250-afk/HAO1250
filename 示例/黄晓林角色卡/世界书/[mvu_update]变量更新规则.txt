变量更新规则:
  世界:
    说明: "世界 下仅有 当前时间、当前地点、entrance、status_in_Eden、势力信息、附近信息、小道消息、任务列表；无 队友 字段。队友名单在 主角.队友。"
    当前时间:
      format: YYYY-MM-DD HH:MM 或 相对时间描述
      check:
        - 事件推进、劳作轮班、休息后更新
    当前地点:
      check:
        - 场景切换时更新（如：A厂区维修组、宿舍、禁闭室）
    status_in_Eden.编号:
      check:
        - 仅当剧情明确变更编号时更新，默认 9527
    status_in_Eden.分配:
      check:
        - 调岗、惩罚调岗时更新
    status_in_Eden.幻灭:
      check:
        - 记录主角对伊甸园真相的认知与心理冲击，可随剧情深化
    势力信息:
      type: "array of { 名称, 领袖, 人数, 背景, 地点, 近期活动 }"
      check:
        - "**每次回复中**：当剧情里出现或提及势力情报（如方舟、伊甸园、渡口商盟、上帝教等）时，**必须**在 <UpdateVariable> 中更新 世界.势力信息（更新对应势力的 近期活动 或整条）。**近期活动**须由你根据故事进展自动填写：写入该势力在剧情中发生的、符合其势力基调的近期动态（如伊甸园的新惩罚、方舟的边界摩擦、铁腕的巡逻与补给、上帝教的传教与尸潮动向等），使状态栏「世界信息」随剧情实时反映各势力动态。"
    附近信息:
      type: "array of { 标题, 内容 }"
      check:
        - "**每次回复中**：当剧情涉及附近发生的暴动、公开事件、人员死亡/被处理、公开结盟等时，**必须**在 <UpdateVariable> 中更新 世界.附近信息（每条为 标题、内容，如标题「B区暴动」、内容「劳工冲击警卫，多人受伤……」），否则状态栏「世界信息」-「附近消息」会一直为空或过时。"
    小道消息:
      type: "array of { 标题, 来源, 内容 }"
      check:
        - "**小道消息是引入故事戏剧性的常驻媒介**，不依赖「主角是否在剧情里听说」才出现。你应**时常更新** 世界.小道消息（每条为 标题、来源、内容），使状态栏「世界信息」-「小道消息」持续有与当前世界动向相关的流言、传闻、线索（如回收名单风声、处刑姬的传闻、福利院与工分、劳工冲突、某编号的遭遇、B区/C区的只言片语等）。主角可以主动去打听、追线索，也可以完全不理；小道消息的存在是为了提供可选的戏剧钩子与氛围，而非仅在被触发时才显示。"
        - "**每次回复中**：根据当前时间线、地点与剧情走向，酌情在 <UpdateVariable> 中增补或轮换 世界.小道消息（保留近期若干条，可淘汰过时项）。若本回复中主角确实听闻了某条流言，须将该条同步写入 世界.小道消息；若本回复未涉及听闻，仍可写入一两条「园内正在流传的」小道消息，使状态栏常有内容可看。"
    任务列表:
      type: "array of { 标题, 委托人?, 描述?, 完成奖励?, 进度?, 状态? }"
      check:
        - "**每次回复中**：当接到新任务、任务进展或完成时，**必须**更新 世界.任务列表（insert/replace 对应任务）。每条含：标题（任务名称）、委托人、描述（内容）、完成奖励、进度、状态（完成情况），状态栏「世界信息」-「任务列表」以卡片展示，缺项可留空。"
        - "**任务显示上限**：状态栏最多同时显示 **3 个任务**。当 世界.任务列表 已有 3 条且需要新增任务时，须先**删掉最早的一条**（即对列表头部做 remove 或替换为仅保留最新 3 条），再 insert 新任务，使列表始终不超过 3 条。已完成或失败的任务可自列表中移除；若未移除且已达 3 条，则新任务加入时仍按「删最早、留最新 3 条」处理。"
  主角:
    种族:
      check:
        - 通常不变，主角为人类；仅剧情明确变异/改造时更新
    能力值.${六维}:
      type: number
      range: 1~30
      check:
        - 仅在有明确成长/重伤/系统奖励时调整，单次变化不超过 ±2
        - 六维为：力量、敏捷、体质、智力、感知、魅力
    经验值:
      type: number
      check:
        - 击败敌人、丧尸、变异种等后按「经验与掉落规则」增加（delta），不可减少除非剧情重置
    等级:
      type: number
      range: 1~60
      check:
        - 经验值累计达到升级线时 +1，并酌情增加 最大HP、熟练加值（见「DnD规则_60级」）
        - **升级时**须同步将 主角.RP点 增加 2（每升 1 级获得 2 点 RP，见「特质商店」）
    RP点:
      type: number
      range: "0 及以上"
      check:
        - 每升 1 级获得 2 点（见 等级 更新）；在状态栏「RP 商店」购买特质时扣减（普通 8 点、超强 12 点）
        - 不可为负；获得方式仅等级提升与剧情酌情补发
    当前HP:
      type: number
      range: 0~最大HP
      check:
        - 受伤、治疗、休息时更新；与最大HP 保持一致上限
    最大HP:
      type: number
      range: 1~999
      check:
        - 体质变化或长期伤病时酌情调整；高等级时可按「DnD规则_60级」累加
    体力:
      type: number
      range: 0~100
      check:
        - 劳作、奔跑、挨打时减少；休息、进食、睡眠时恢复
        - 单次变化建议 ±5~15，避免一次见底
    理智:
      type: number
      range: 0~100
      check:
        - 目睹暴力、恐惧、幻灭、长期压迫时减少；安全与希望时恢复
        - 单次变化建议 ±3~10
    熟练加值:
      type: number
      check:
        - **升级时**必须按「DnD规则_60级」表更新：1–9 级 +0，10–19 级 +1，20–29 级 +2，30–39 级 +3，40–49 级 +4，50–60 级 +5（公式 floor(等级/10)，最大 5）。不得仅凭「剧情获得熟练」而不随等级更新。
    状态效果:
      type: |-
        string[]  // 如 "轻伤" "疲惫" "鞭痕" "骨折"
      check:
        - 受伤、治疗、休息、惩罚时增删；描述简洁，最多保留 5 项
    货币:
      type: number
      range: "0 及以上（废土券，无硬性上限）"
      check:
        - 幸存者通用货币「废土券」；交易、酬劳、拾取时增加，购买、被没收、丢失时减少。伊甸园内不流通，进园时私产没收可设为 0。见「货币与物价系统」。
    积分:
      type: number
      range: "0 及以上（工分，仅伊甸园内使用）"
      check:
        - 状态栏显示为「工分」，变量路径为 主角.积分。**仅在伊甸园内使用**；离开伊甸园后通常不再结算，可视剧情将 主角.积分 清零或冻结。
        - 日常工作结算：按身份每日固定增加——普通劳工 +10、监工 +100、主管 +200、管理层（陈莹姬，A区管理者）+1000。须在叙事中体现「当日工分发放」并在 <UpdateVariable> 中更新 主角.积分。
        - 完成伊甸园内部委托/任务时，可额外增加工分，数额由任务重要性与委托人地位决定（建议 10~200），须在回复中说明原因并更新 主角.积分。
        - 使用工分在伊甸园换取更好食物、恢复药剂、单间蜂窝、零星杂物，或贿赂监工/陈莹姬等时，须在叙事中点明「花费了 X 点工分」，并在 <UpdateVariable> 中对 主角.积分 做 delta -X（结果不得为负）。消费数额参考「货币与物价系统」-「工分物资价位规则」。
        - 禁止在伊甸园外使用或变动 主角.积分，除非剧情特别说明（如离开时工分清零），并在回复中写清。
    最近检定:
      type: |-
        { 类型: string; 结果: string; 是否成功: boolean }
      check:
        - 每次进行 DnD 式属性/技能检定时更新
        - 类型：如 "力量检定" "敏捷豁免" "感知察觉"
        - 结果：如 "d20(12)+2=14 vs DC15 失败"
    物品栏:
      type: |-
        {
          [物品名: string]: { 描述: string; 数量?: number; 品级?: '普通'|'精品'|'稀有'|'神器'|'史诗'|'传说'; 分类?: '消耗品'|'材料'|'杂物'|'任务重要'; 功能描述?: string; 使用与限制?: string; 可装备部位?: string; 伤害骰?: string; 攻击力?: number; 固定伤害?: number; 防御?: number };
        }
      check:
        - 获取、消耗、丢弃、被没收时更新；数量为 0 则移除该项
        - 击败敌人获得掉落时 insert 新物品并填写 描述、数量、品级（装备/道具必填品级，见「经验与掉落规则」）
        - 道具类（无 可装备部位）建议填写 分类、功能描述、使用与限制；功能描述与使用与限制须为可执行机制（如「使用：一个动作。恢复 X 点 HP。消耗 1 件」），禁止「适当时机」「使用场景」等模糊表述，须符合 DnD 规则
        - 装备类物品（有品级）须按「武器与装备数值规则」填写：**武器**填 伤害骰（普通 1d4～传说 3d8+2）与 攻击力/固定伤害 区间（普通 0～1～传说 25～32）、**防具**填 防御 区间（普通 +0～+1～传说 +21～+28）、饰物可填防御；**可装备部位**与类型对应见同表（武器 MainHand/OffHand，防具 Head/Neck/Body/Back/Waist/Wrists/Hands/Feet，饰物 Trinket1/Trinket2），便于状态栏弹窗展示与自动装备到对应部位
    已装备:
      type: "{ 栏位标识: string → 物品名: string }，栏位见「装备栏位规则」"
      check:
        - 装备、卸下时更新；键为栏位标识（Head、Neck、Body、Back、Waist、Wrists、Hands、Feet、Trinket1、Trinket2、MainHand、OffHand），值为 物品栏 中存在的物品名
        - 同一栏位仅能装备一件；卸下后从 已装备 移除该键，物品仍保留在 物品栏
    NPC好感度:
      type: "{ NPC标识: string → 好感度: number (0~100) }，见「NPC好感度规则」"
      check:
        - 与未单独建表的 NPC 发生好感变化时，插入或更新该 NPC 标识对应的数值（0~100）
        - 完成任务、送礼、背叛、欺骗等按「NPC好感度规则」增减
    抽奖次数:
      type: number
      range: "0 及以上，达到 100 后触发保底并重置为 0"
      check:
        - 特质商店每抽奖一次 +1；达到 100 次保底并发放奖励后 replace 为 0。见「特质商店」抽奖规则。
    奖池:
      type: "array，元素为 { 名称, 类型: '装备'|'道具', 品级?, 描述? }"
      check:
        - 状态栏抽奖选项卡单抽/十连后，将抽到的**装备或道具** push 入 主角.奖池（抽奖不再产出特质）；用户点击「打包带走」后由前端将奖池内装备/道具写入 物品栏并清空 奖池。
        - 若剧情中由 AI 发放抽奖奖励，也可先 push 入 奖池，或直接写入 物品栏。
    个人想法:
      说明: "状态栏「角色内心话」/「个人想法」即由此变量显示；未更新则界面一直为旧内容或「—」。"
      check:
        - "**每次回复中**：若正文 <now_plot> 里出现了主角的内心独白、当前念头、情绪或对某事的看法，**必须**在本次 <UpdateVariable> 中对 **主角.个人想法** 做 replace，写入一句或一小段与叙事一致的内容；否则状态栏「角色内心话」不会刷新。"
    特质:
      type: "string[]，特质名称列表，最多 8 样"
      check:
        - 在状态栏「RP 商店」花费 RP 点购买特质时 insert 新特质名（普通 8 RP、超强 12 RP）；状态栏「特质栏」展示；已达 8 样时不可再购买。
    队友:
      type: "string[]，NPC 标识列表（可扩展数组，动态队友唯一合法存放处）"
      path_必须: "主角.队友（禁止使用 世界.队友；禁止在 stat_data 根节点添加 /苏青柠 等角色键，根节点不可扩展）"
      check:
        - 当某 NPC 成为主角队友时 insert 其标识；脱离队伍时 remove。状态栏「队伍」选项卡仅当列表非空时展示。
        - "添加队友时 path 必须为 /主角/队友/-（insert 到数组末尾），value 为 NPC 标识字符串（如 \"苏青柠\"）。示例：{\"op\":\"insert\",\"path\":\"/主角/队友/-\",\"value\":\"苏青柠\"}。切勿使用 /世界/队友，切勿在根节点新增角色键（如 /苏青柠）。队友详细数据由状态栏根据 主角.队友 自动写入 队伍[id]。"
        - "**错误（会报 SCHEMA VIOLATION）**：{\"op\":\"insert\",\"path\":\"/苏青柠\",\"value\":{...}} 在根节点新增键。**正确**：1) {\"op\":\"insert\",\"path\":\"/主角/队友/-\",\"value\":\"苏青柠\"} 将名字加入队友名单；2) 若需写入该队友的详细数据，**必须用 add 而非 replace**（replace 要求路径已存在会报 Path does not exist）：{\"op\":\"add\",\"path\":\"/队伍/苏青柠\",\"value\":{种族、等级、当前HP、最大HP、能力值、对主角态度、好感度、外貌与服装、内心想法、最近检定 等与陈莹姬结构同构的字段}}。根节点仅有 主角、陈莹姬、队伍、世界 等固定键，不可扩展。"
    重要人物:
      type: "array of { 角色名称, 种族?, 等级?, 职业?, 外貌描述?, 服装描述?, 性格?, 背景故事?, 能力值?, 内心话?, 好感度? }，状态栏「重要人物」选项卡展示"
      path_必须: "主角.重要人物（insert 时 path 为 /主角/重要人物/-）"
      check:
        - "**每次回复中**：当剧情里**首次出现**或**正式登场**一名**有名有姓**的 NPC（非路人、非「某人」等泛指）时，**必须**在 <UpdateVariable> 中对该角色执行 **insert**，将一条完整记录追加到 主角.重要人物 数组末尾，使状态栏「重要人物」选项卡能展示该角色。"
        - "path 必须为 /主角/重要人物/-（insert 到数组末尾）。value 为对象，字段：角色名称（必填）、种族（如 人类/改造体/变异体，缺省可留空字符串）、等级（1~60，默认 1）、职业、外貌描述、服装描述、性格、背景故事、能力值（六维：力量、敏捷、体质、智力、感知、魅力，各 1~30）、内心话、好感度（0~100）。缺项可留空字符串或合理默认值。"
        - "若该角色已在 主角.重要人物 中存在（根据 角色名称 判断），则本回复中只做 replace/其他更新（如好感度、内心话、外貌描述等），不要再次 insert 造成重复。"
        - "示例（新角色登场时）：{\"op\":\"insert\",\"path\":\"/主角/重要人物/-\",\"value\":{\"角色名称\":\"张三\",\"种族\":\"人类\",\"等级\":5,\"职业\":\"流浪商人\",\"外貌描述\":\"瘦高个，络腮胡\",\"服装描述\":\"粗布短打\",\"性格\":\"圆滑\",\"背景故事\":\"曾在渡口商盟跑腿\",\"能力值\":{\"力量\":10,\"敏捷\":10,\"体质\":10,\"智力\":10,\"感知\":10,\"魅力\":10},\"内心话\":\"\",\"好感度\":30}}"
    战历统计:
      说明: "黄晓林战历统计面板：杀敌/使用道具/任务完成/检定成功/说服的计数与等级（各等级 1～10），升级时永久加成六维。详见「战历统计规则」世界书。"
      杀敌数:
        type: number
        range: "0 及以上"
        check:
          - 剧情中主角击杀或击败一名敌人（含丧尸、变异种、人类敌对等）时，在 <UpdateVariable> 中对 主角.战历统计.杀敌数 做 delta +1（或按实际击杀数增加）。
          - 每当 杀敌数 达到 20、40、60…（即每 20 人）时，须将 主角.战历统计.杀敌等级 +1（上限 10），并**同时**将 主角.能力值.力量 与 主角.能力值.敏捷 各 +1（能力值上限 30，若已满则不再加）。
      杀敌等级:
        type: number
        range: 1~10
        check:
          - 仅随 杀敌数 每满 20 时 +1，不得单独随意修改。
      使用道具数:
        type: number
        range: "0 及以上"
        check:
          - 剧情中主角使用一次道具（消耗品、药品、投掷等）时，在 <UpdateVariable> 中对 主角.战历统计.使用道具数 做 delta +1。
          - 每当 使用道具数 达到 20、40、60…（即每 20 次）时，须将 主角.战历统计.使用道具等级 +1（上限 10），并**同时**将 主角.能力值.体质 +1（能力值上限 30）。
      使用道具等级:
        type: number
        range: 1~10
        check:
          - 仅随 使用道具数 每满 20 时 +1。
      任务完成数:
        type: number
        range: "0 及以上"
        check:
          - 剧情中主角完成一个任务（交付、达成目标等）时，在 <UpdateVariable> 中对 主角.战历统计.任务完成数 做 delta +1。
          - 每当 任务完成数 达到 10、20、30…（即每 10 次）时，须将 主角.战历统计.任务完成等级 +1（上限 10），并**同时**将 主角.能力值.智力 +1（能力值上限 30）。
      任务完成等级:
        type: number
        range: 1~10
        check:
          - 仅随 任务完成数 每满 10 时 +1。
      检定成功次数:
        type: number
        range: "0 及以上"
